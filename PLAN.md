# PLAN.md
# MediAssist AI - 프로젝트 계획서

---

## 1. 프로젝트 개요

### 1.1 목표
ML 기반 증상 분석과 RAG 기반 의학 문헌 검색을 결합한
의료 진단 보조 시스템 구축

### 1.2 범위
```
┌─────────────────────────────────────────────────────────────────┐
│                      프로젝트 범위                               │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    In Scope (Phase 1)                    │   │
│  │  ✅ 증상 기반 질병 분류 ML 모델                           │   │
│  │  ✅ 의료 이미지 분석 (흉부 X-ray)                        │   │
│  │  ✅ RAG 기반 의학 문헌 검색                              │   │
│  │  ✅ 진단 보조 리포트 생성                                │   │
│  │  ✅ 웹 기반 사용자 인터페이스                            │   │
│  │  ✅ 사용자 인증/인가                                     │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │                    Out of Scope                          │   │
│  │  ❌ 최종 진단 결정                                       │   │
│  │  ❌ 처방 자동화                                          │   │
│  │  ❌ EMR 직접 연동 (Phase 2)                              │   │
│  │  ❌ 모바일 앱 (Phase 2)                                  │   │
│  │  ❌ 실시간 환자 모니터링                                 │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 성공 지표 (KPIs)
| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 증상 분류 정확도 | ≥ 85% | 검증 데이터셋 |
| 문헌 검색 적합도 | ≥ 90% | 의사 평가 |
| 시스템 응답 시간 | < 30초 | 모니터링 |
| 사용자 만족도 | ≥ 4.0/5.0 | 설문 |
| 서비스 가용성 | ≥ 99.9% | 모니터링 |

| ID | 리스크 | 확률 | 영향 | 완화 전략 |

```
R1: ML 정확도 미달
├── 예방: 다양한 데이터셋 확보, 정기 모델 평가

└── 비상: 규칙 기반 백업 시스템

└── 비상: 기능 축소 출시

R6: 데이터 보안 사고
├── 예방: 암호화, 접근 통제, 감사 로그
```
